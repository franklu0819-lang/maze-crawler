# 迷宫爬行（Crawl）

一款带战争迷雾的迷宫策略游戏。两名玩家在不断向北滚动的迷宫中导航，建造机器人来探索、收集能量，并比对手存活更久。

## 概述

每名玩家在 20×20 迷宫底部附近以一个**工厂**机器人开始。迷宫随时间向北滚动——南边界不断推进，摧毁所有被留在后面的东西。最后仍有工厂存活的玩家获胜。

迷宫具有**东西对称性**：左半部分是右半部分的镜像，两侧之间偶尔有通道门相连。两名玩家分别在对称的两侧开始。

## 机器人类型

| 类型 | 费用 | 最大能量 | 移动周期 | 视野 | 特殊能力 |
|------|------|----------|----------|------|----------|
| **工厂** | — | 无限 | 2 回合 | 4 | 建造单位、跳跃（20 回合冷却）、不可摧毁 |
| **侦察兵** | 50 | 100 | 1 回合 | 5 | 快速探索者 |
| **工人** | 200 | 300 | 2 回合 | 3 | 建造/拆除墙壁（100 能量） |
| **矿工** | 300 | 500 | 2 回合 | 3 | 在采矿节点上转化为能量矿场 |

所有机器人每回合消耗 1 能量。能量为 0 的机器人被强制闲置。

## 动作

每回合，你返回一个将机器人 UID 映射到动作字符串的字典。

### 移动
- `NORTH`、`SOUTH`、`EAST`、`WEST` — 向该方向移动一格（被墙壁阻挡）。**成功移动出棋盘北边界或南边界的单位将被摧毁（无墙壁阻挡时）。** 东西方向总是被边界墙阻挡。

### 工厂动作
- `BUILD_SCOUT`、`BUILD_WORKER`、`BUILD_MINER` — 在工厂**正北**的格子生成新机器人。要求工厂与生成格之间无墙。建造冷却 10 回合。新机器人在移动阶段*之前*放置，因此在战斗中算作静止单位——如果该格上的敌方单位同一回合移走，新机器人安全着陆；否则在生成格上进行碾压战斗结算。
- `JUMP_NORTH`、`JUMP_SOUTH`、`JUMP_EAST`、`JUMP_WEST` — 向一个方向跳跃 2 格，无视所有墙壁。跳跃一定会执行并消耗冷却。**如果落点在棋盘外，工厂被摧毁。** 20 回合冷却。

### 工人动作
- `BUILD_NORTH`、`BUILD_SOUTH`、`BUILD_EAST`、`BUILD_WEST` — 在工人所在格与该方向相邻格之间添加墙壁。消耗 100 能量。工人存活。
- `REMOVE_NORTH`、`REMOVE_SOUTH`、`REMOVE_EAST`、`REMOVE_WEST` — 移除工人所在格与该方向相邻格之间的墙壁。消耗 100 能量。工人存活。

**固定墙壁（不可修改）：** 外围边界（最左列的东/西和最右列的东/西）和中央对称轴（第 `width/2 - 1` 列的东和第 `width/2` 列的西）。对固定墙壁执行 BUILD/REMOVE，或墙壁已处于目标状态时，仍消耗 100 能量但无效果。固定墙壁在可视化器中显示为**双线**。

### 矿工动作
- `TRANSFORM` — 销毁矿工并在其位置创建能量矿场。**要求矿工站在采矿节点上。** 消耗 100 能量。矿场接收矿工的剩余能量（不超过矿场上限）。

### 其他
- `TRANSFER_NORTH`、`TRANSFER_SOUTH`、`TRANSFER_EAST`、`TRANSFER_WEST` — 将所有能量发送给相邻的友方机器人。被墙壁阻挡。目标能量上限为其最大值（工厂无上限）。
- `IDLE` — 什么都不做。

## 战斗

当两个或更多机器人在同一回合结束时处于同一格子时，适用碾压规则——**不分敌我，友军伤害有效。**

- **碾压等级：** 工厂 > 矿工 > 工人 > 侦察兵。较强的类型摧毁较弱的。
- **同类型：** 该类型的所有机器人全部摧毁。两个友方侦察兵走到同一格子会互相湮灭。
- **工厂：** 对任何非工厂单位（友方或敌方）不可摧毁，并碾压它们。两个敌方工厂在同一格子互相摧毁（游戏结束，参见胜利条件）。
- **战斗格上的水晶：** 存活的机器人（如果有）收集水晶能量。如果没有机器人存活，水晶被消耗。

在已占据的格子上生成机器人会正常触发战斗——包括生成格子被己方单位占据时的友军伤害。

## 地图要素

### 水晶
散布在整个迷宫中（每格 6% 密度）。任何移动到水晶上的机器人收集其能量（10-50）。水晶仅在视野范围内可见，离开视野后不再被记忆。

### 采矿节点
稀有位置（3% 密度），在地图上标记，矿工可以在其上转化为矿场。当矿场创建在采矿节点上时，该节点被消耗。采矿节点永远不会与水晶重叠。

### 矿场
由矿工在采矿节点上使用 TRANSFORM 创建。矿场每回合产生 **50 能量**，最大储存 **1000 能量**。站在矿场上的友方机器人从中收集能量。矿场一旦被发现就会被永久记忆（即使在视野范围外）。

## 战争迷雾

每个机器人有一个视野范围（曼哈顿距离）。你只能看到所有己方机器人联合视野内的内容。

| 数据 | 视野内可见 | 离开视野后记忆 |
|------|-----------|---------------|
| 墙壁/布局 | 是 | 是（永久） |
| 水晶 | 是 | 否 |
| 敌方机器人 | 是 | 否 |
| 己方机器人 | 始终 | 不适用 |
| 矿场（任意所有者） | 是 | 是（永久，最后已知状态） |
| 采矿节点 | 是 | 否 |

## 迷宫滚动

南边界随时间推进，摧毁所有在其下方的机器人、矿场和水晶。

- **初始：** 每 4 回合滚动一次
- **加速：** 在 400 步内线性加速
- **最终：** 从第 400 步起每回合滚动一次（直到第 500 步游戏结束）

如果工厂落在南边界以下，该玩家被淘汰。

## 回合处理顺序

1. **冷却计时** — 减少移动、跳跃和建造冷却
2. **动作验证** — 验证动作合法性
3. **能量消耗** — 每个机器人失去 1 能量；能量为 0 的机器人被强制闲置
4. **特殊动作** — TRANSFORM（矿工）、BUILD_DIR/REMOVE_DIR（工人墙壁）、BUILD_SCOUT/WORKER/MINER（工厂）、TRANSFER（按此顺序）
5. **移动 + 战斗** — 同时移动，然后结算碰撞
6. **水晶收集** — 在水晶格上的机器人收集能量
7. **矿场能量填充** — 在友方矿场上的机器人收集能量
8. **矿场产能** — 每个矿场获得 50 能量（上限 1000）
9. **滚动推进** — 推进边界，生成新行，放置水晶/节点
10. **边界摧毁** — 摧毁南边界以下的机器人/矿场
11. **胜利条件检查**
12. **更新观测** — 计算战争迷雾，构建每位玩家的视图

## 胜利条件

1. **生存：** 如果一方工厂被摧毁（被滚动到边界以下），另一方获胜。
2. **同时淘汰：** 如果双方工厂在同一回合被摧毁（例如工厂互撞，或同时被滚出），适用**决胜级联**。
3. **时间限制（第 500 步）：** 如果双方工厂仍然存活，适用**决胜级联**。

### 决胜级联

1. **总能量**（所有机器人）— 较高者获胜
2. **单位数量**（所有机器人）— 较多者获胜
3. **真正的平局** — 双方获得奖励 `0.5`

## 奖励

- **存活（游戏中）：** 所有己方机器人的总能量
- **通过决胜级联获胜：** `1`
- **通过决胜级联失败：** `0`
- **平局：** `0.5`
- **被淘汰（对手存活）：** `淘汰步数 - 总步数 - 1`（负值）；获胜者获得总能量

## 观测格式

```python
def agent(obs, config):
    obs.player        # 你的玩家索引（0 或 1）
    obs.walls         # 扁平数组：索引 = (行 - 南边界) * 宽度 + 列
                      # 值：墙壁位域，-1 = 未发现
    obs.crystals      # {"列,行": 能量} — 仅当前可见
    obs.robots        # {"uid": [类型, 列, 行, 能量, 所有者, 移动冷却, 跳跃冷却, 建造冷却]}
    obs.mines         # {"列,行": [能量, 最大能量, 所有者]} — 一旦发现则被记忆
    obs.miningNodes   # {"列,行": 1} — 仅当前可见
    obs.southBound    # 当前南边界行
    obs.northBound    # 当前北边界行
```

### 墙壁位域

```
北 = 1, 东 = 2, 南 = 4, 西 = 8
```

检查墙壁：`if wall_value & 1:` 表示有北墙。固定墙壁（外围和中央对称轴）具有相同的位域表示但不能被工人修改；可视化器将其渲染为双线。

## 快速入门

```python
from kaggle_environments import make

def my_agent(obs, config):
    actions = {}
    for uid, data in obs.robots.items():
        rtype, col, row, energy, owner = data[0], data[1], data[2], data[3], data[4]
        if owner != obs.player:
            continue
        if rtype == 0:  # 工厂
            if energy >= config.workerCost:
                actions[uid] = "BUILD_WORKER"
            else:
                actions[uid] = "NORTH"
        else:
            actions[uid] = "NORTH"
    return actions

env = make("crawl", configuration={"randomSeed": 42})
env.run([my_agent, "random"])
env.render(mode="ipython", width=800, height=800)
```
